When a new ETHGlobal hackathon concludes, you need to add its projects to the database. Currently, this requires running dated update scripts in the `add_new_projects_*/` directories.

<Warning>
  This is a manual process with separate scripts for each update. The goal is to create a single script that can update by event name - see [future improvements](#future-improvements).
</Warning>

## Update workflow

The process for adding new hackathon projects follows these steps:

<Steps>
  <Step title="Scrape new projects">
    Run the scraping script to get projects from recent hackathons
  </Step>
  <Step title="Upload new prizes (if needed)">
    Add any new sponsor prizes that don't exist in the database
  </Step>
  <Step title="Upload new projects">
    Insert projects for the specific event into the database
  </Step>
  <Step title="Update existing projects">
    Add new prize relationships to projects that won newly added prizes
  </Step>
</Steps>

## Example: October 2024 update

The October 24 update added projects from ETHGlobal New York 2025 and Unite. This is a three-script process:

### 1. Scrape new projects

[scrape_ny_and_unite_projects.py](/home/daytona/workspace/source/scripts/add_new_projects_oct_24/scrape_ny_and_unite_projects.py) scrapes projects from recent pages:

- **Cloudflare handling**: Detects and waits for Cloudflare challenges

### 2. Upload new projects

[upload_ny_and_unite_projects.py](/home/daytona/workspace/source/scripts/add_new_projects_oct_24/upload_ny_and_unite_projects.py) filters for specific events:

```python
def upload_new_projects():
    # Read the JSON file
    with open('new_ny_and_unite_projects.json', 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    # Filter projects for "ETHGlobal New York 2025" event
    all_projects = data['projects']
    new_projects = [project for project in all_projects if project['event'] == 'ETHGlobal New York 2025']
    
    total = len(new_projects)
    print(f"Found {total} new projects to upload")
```

**Event filtering**: Only uploads projects where `event == 'ETHGlobal New York 2025'`

### 3. Update existing projects with new prizes

[update_projects_with_new_prizes.py](/home/daytona/workspace/source/scripts/add_new_projects_oct_24/update_projects_with_new_prizes.py) adds prize relationships to existing projects:

```python
def load_new_prize_urls():
    """Load the prize URLs from new_prizes_oct_25.json"""
    with open('../add_new_prizes/new_prizes_oct_25.json', 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    prize_urls = [prize['img_url'] for prize in data['prizes']]
    return prize_urls

def update_projects_with_new_prizes():
    # Load new prize URLs
    new_prize_urls = load_new_prize_urls()
    
    # Find projects that have any of the new prize URLs
    projects_to_update = []
    for project in target_projects:
        project_prize_urls = project.get('prizes', [])
        matching_prizes = [url for url in project_prize_urls if url in new_prize_urls]
        if matching_prizes:
            projects_to_update.append(project)
```

This script:

1. Loads a list of newly added prize URLs
2. Finds projects that won any of those prizes
3. Creates relationships in the `project_prizes` junction table
4. Skips relationships that already exist

## Duplicate prevention

All update scripts check for existing projects:

```python
# Check if project already exists
existing = supabase.table('projects').select('id').eq('url', project['url']).execute()
if existing.data and len(existing.data) > 0:
    print(f"Project already exists: {project['title']}")
    continue
```

This prevents duplicate entries when re-running scripts.

## Prize relationship checking

The update script also checks for existing prize relationships:

```python
# Check if relationship already exists
existing_relation = supabase.table('project_prizes')
    .select('id')
    .eq('project_id', project_id)
    .eq('prize_id', prize_id)
    .execute()

if not existing_relation.data or len(existing_relation.data) == 0:
    # Create the relationship
    supabase.table('project_prizes').insert({
        'project_id': project_id,
        'prize_id': prize_id
    }).execute()
```

## Handling unknown prizes

Update scripts warn about missing prizes instead of creating unnamed entries:

```python
def get_or_create_prize(img_url):
    result = supabase.table('prizes').select('id').eq('img_url', img_url).execute()
    if result.data and len(result.data) > 0:
        return result.data[0]['id']
    
    # For unknown prizes, skip them and log a warning
    print(f"Warning: Unknown prize image URL: {img_url}")
    print("Skipping this prize to avoid creating unnamed prize entries.")
    return None
```

<Note>
  If you see warnings about unknown prizes, you need to add them to the prizes table first using a script similar to `01_upload_prizes.py`.
</Note>

## Running an update

To add projects from a new hackathon:

```bash
cd source/scripts/add_new_projects_oct_24

# Step 1: Scrape recent projects
python scrape_ny_and_unite_projects.py

# Step 2: Upload projects for the target event
python upload_ny_and_unite_projects.py

# Step 3: Update existing projects with new prizes
python update_projects_with_new_prizes.py
```
