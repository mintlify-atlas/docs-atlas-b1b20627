This guide will help you set up the ETHGlobal Explorer project on your local development environment.

## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js 20 or higher
- npm (comes with Node.js)
- Python 3.x (required for running scraping scripts)
- pip (Python package manager)

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the ETHGlobal Explorer repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd ethglobal-explorer
    ```
  </Step>

  <Step title="Install web app dependencies">
    Navigate to the web-app directory and install the required Node.js dependencies:
    
    ```bash
    cd web-app
    npm install
    ```
    
    This will install the following key dependencies:
    - Next.js 15.1.9 (React framework)
    - React 19.0.0
    - Supabase JS client 2.47.10
    - Vercel Analytics
    - TypeScript and Tailwind CSS
  </Step>

  <Step title="Install Python dependencies">
    If you plan to run the scraping scripts, install the Python dependencies:
    
    ```bash
    cd ../scripts
    pip install -r requirements.txt
    ```
    
    This will install:
    - selenium (web scraping)
    - beautifulsoup4 (HTML parsing)
    - supabase (database client)
    - python-dotenv (environment variables)
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the `web-app` directory with your Supabase credentials. See the [Environment Variables](/development/environment-variables) page for details.
    
    ```bash
    cd ../web-app
    touch .env
    ```
  </Step>

  <Step title="Run the development server">
    Start the Next.js development server:
    
    ```bash
    npm run dev
    ```
    
    The application will be available at [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Available scripts

### Web app

<CodeGroup>
```bash npm run dev
npm run dev
```

```bash npm run build
npm run build
```

```bash npm start
npm start
```

```bash npm run lint
npm run lint
```
</CodeGroup>

- `npm run dev` - Starts the development server with hot reload
- `npm run build` - Creates an optimized production build
- `npm start` - Runs the production server (requires running `build` first)
- `npm run lint` - Runs ESLint to check code quality

### Python scripts

The `scripts` directory contains several Python scripts for scraping and uploading data:

- `01_upload_prizes.py` - Uploads prize data to the database
- `02_scrape_projects.py` - Scrapes project data from ETHGlobal showcase
- `03_upload_projects.py` - Uploads scraped project data to the database
- `04a_upload_hackathons.py` - Uploads hackathon event data
- `05_scrape_hackathons.py` - Scrapes hackathon information

<Note>
The scraping scripts require valid Supabase credentials in your environment. Make sure to set up your `.env` file with `SUPABASE_URL` and `SUPABASE_SERVICE_KEY` before running these scripts.
</Note>

## Development workflow

1. Make your changes to the code
2. The Next.js dev server will automatically reload with your changes
3. Test your changes at `http://localhost:3000`
4. Run `npm run lint` to check for code quality issues
5. Run `npm run build` to ensure your changes build successfully

## Troubleshooting

### Port 3000 already in use

If port 3000 is already in use, you can specify a different port:

```bash
npm run dev -- -p 3001
```

### Environment variables not loading

Ensure your `.env` file is in the `web-app` directory (not the root) and that you've restarted the development server after adding environment variables.

### Python dependencies installation fails

Make sure you're using Python 3.x and pip is up to date:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```
