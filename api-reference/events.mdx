title: GET /api/events
description: Retrieve ETHGlobal hackathon events with search and pagination

Retrieve a paginated list of ETHGlobal hackathon events. Supports search by event name.

## Endpoint

```
GET /api/events
```

## Query parameters

All query parameters are optional.

<ParamField query="page" type="number" default="1">
  Page number for pagination. Each page returns up to 100 events.
</ParamField>

<ParamField query="q" type="string">
  Search term to filter events by name. Uses case-insensitive pattern matching.
</ParamField>

## Response

<ResponseField name="data" type="array">
  Array of event objects matching the query criteria.
  
  <Expandable title="Event object">
    <ResponseField name="id" type="number">
      Unique identifier for the event.
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Event name or identifier (e.g., `ethonline2024`, `ethglobal-singapore-2024`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totalCount" type="number">
  Total number of events matching the query criteria (across all pages).
</ResponseField>

## Implementation details

The endpoint implements the following behavior (from route.ts:72-105):

- Returns 100 items per page
- Search term (`q`) uses case-insensitive ILIKE pattern matching on event names
- Results are paginated using range-based queries

## Use cases

This endpoint is particularly useful for:

- Populating event filter dropdowns in UI components
- Discovering available events for project filtering
- Building event-specific views or analytics
