## How prize filtering works

Prize filtering uses a junction table architecture to handle the many-to-many relationship between projects and prizes. A single project can win multiple prizes, and each prize can be awarded to multiple projects.

### Database query flow

```typescript
if (selectedPrize) {
  // Step 1: Query the junction table for matching project IDs
  const { data: projectIds } = await supabase
    .from('project_prizes')
    .select('project_id, prize:prize_id!inner(name)')
    .eq('prize.name', selectedPrize);
  
  // Step 2: Filter main query to only include those project IDs
  if (projectIds) {
    const ids = projectIds.map(p => p.project_id);
    query = query.in('id', ids);
    countQuery = countQuery.in('id', ids);
  }
}
```

This two-step process:
1. Finds all project IDs that have won the selected prize
2. Filters the main projects query to only include those IDs

### Data structure

The prize data is structured across three tables:
- **projects**: Core project information
- **prizes**: Prize details (name, img_url)
- **project_prizes**: Junction table linking projects to prizes

## Prize display

Prizes are visually represented in the UI using sponsor logos:

```typescript
<Image 
  src={prize.img_url}
  alt={prize.name}
  title={prize.name}
  className="h-6 w-6 object-contain rounded-full"
  width={24}
  height={24}
/>
```

Each prize logo is 24x24 pixels and appears in the "Prizes" column of the projects table. Hovering over a logo displays the full sponsor name.

## Prize categories

Sponsors typically offer different types of prizes:
- **Grand/Pool Prizes**: Open to all participants
- **Bounties**: Specific technical challenges
- **Best Use**: Integration awards for using sponsor technology
- **Finalist**: Top overall projects selected by ETHGlobal

<Note>
The "Finalist" prize is awarded by ETHGlobal to top projects at each event and represents exceptional overall quality.
</Note>
